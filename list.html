Restaurant List App
The application was done under my apprenticeship for a Senior Developer specializing in the .NET Framework.

It’s basic concept is to leverage .NET, specifically Active Server Pages (ASP) to dynamically render changes to a directory of restaurants on a browser. A user when encountering a new restaurant can add, edit, or delete the entity as well as include a short description.

User input is stored through SQL Server, a relational database management system (RDBMS) on the back-end. Accordingly, I abstracted the need for hard-written queries into the database by using an Object Relational Mapper (ORM) Entity Framework and Language Integrated Query (LINQ). In short, these components allowed me to use C# an Object Oriented Programming Language (OOP) to work directly with the database.

The Model View Controller (MVC) design pattern was used to construct relevant data models. These models were templated and viewable to the user through Razor markup. In regard to networking, HTTP requests were handled by routing URL file paths to specified controllers.

Ideally, I’d like the application to evolve in scope. I would like it to handle a more robust input of data. This would allow me to work with data relationships like, One-to-One, One-to-Many, or Many-to-Many. The intent being to organize said data in a logically sound manner, while preserving it’s integrity and simplifying accessibility.
