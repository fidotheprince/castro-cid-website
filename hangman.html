Hangman App
After completing the restaurant list project, I kept learning from my mentor and watched him and another Senior Developer create a Command Line Interface (CLI) application of Hangman with Ruby. The two seasoned Developers went from knowing zero about Ruby to programming something functional within an amalgamation of less than 48 hours. During this process I asked multiple questions, contributed valuable input, and most importantly, took the time to understand the logical concepts applied by the duo.

Upon finishing the application, I had a deep desire to elevate what I had learned to a new level. Instead of a CLI, I wanted a single-page Web Application. Hangmantwosix (take a peek) is a result of this inspiration.

The JavaScript (JS) for the application is written to evaluate the current state of the game and send that given state to be visually rendered for the user. To do this, the initial string data type representing the hidden word is made into an array data structure. This allows JS to iterate through each letter item and assess whether it matches a user guess or not. Accordingly, counts are kept for correct and incorrect user guesses. These counts help assess when the game has reached an end state. Moreover, each guess changes a key:value pairing within an object representing the hidden word. The key in this case would be the letter and the value being a boolean data type. A letter's true or false value gives JS the information to know which letters to render visually (true) and which letters to hide (false).

I think this application can improve by organizing it’s logic in a more efficient manner. I used my understanding of both Function Programming and Object Oriented Programming for it’s code. However, the consequence is that it lacks much needed structure. Future programs will be written by choosing one of these paradigms instead of straddling both.
